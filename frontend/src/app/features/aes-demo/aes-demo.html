หลักการทำงาน

  1. PBKDF2: แปลง passphrase + salt → AES-256 key (slow function, กัน brute-force)
  2. Salt 16 bytes สุ่ม: ทำให้ passphrase เดียวกัน → key ต่างกัน (กัน rainbow table)
  3. IV 12 bytes สุ่ม: เพื่อให้ ciphertext ของ plaintext เดียวกัน ออกมาไม่เหมือนกันทุกครั้ง
  4. Auth Tag 16 bytes: เพื่อตรวจสอบความถูกต้อง — ถ้า ciphertext ถูกแก้ไข decrypt จะ fail
  5. Packed format: salt(16) || iv(12) || ciphertext || tag(16) → Base64 → 1 string
